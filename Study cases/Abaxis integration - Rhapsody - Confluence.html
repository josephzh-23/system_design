Integration design overview

We have a dedicated service for Abaxis integration: abaxis-connector. It is responsible for all communications between Rhapsody and Abaxis API and for converting Rhapsody data formats to Abaxis data formats and vice versa.

 

Responsibilities of the service:

  • provides ability to switch clinic integration on and off

  • provides ability to place order via Abaxis API (exposes endpoint for being called by other services, now it is used by apigw rhapsody)

  • pulls lab results from Abaxis API if available (order statuses “PARTIAL RESULTS“ and “RESULTS RECEIVED“ indicate that we need to try to get results from Abaxis API)

  • provides an ability to synchronize lab test catalog with Abaxis catalog got from Abaxis API (exposes an endpoint which is called by scheduled job every Sunday)

Integration switching on/off

Abaxis lab integration can be switched on in Admin > Practices > Practice details > “Lab intagration“ section.

To switch Abaxis integration on, practice needs to provide client ID. This info is given to the practice by Abaxis.

When user presses “Test credentials“ button, portal sends a call to backend. Backend validates the request and saves credentials if they pass validation. The integration is switched on.

Practice integration credentials cannot be used by another practice

If provided credentials (client ID) are not valid, backend doesn’t save them, and user will see a warning under credentials input fields.

Rhapsody has clinic “Abaxis proxy end-to-end 123“ in staging for testing Abaxis integration.

Abaxis integration for Happy Vet is off on staging.

Controller: vet.petabyte.abaxis.controller.BusinessSettingsController
DB table: abaxis_settings

Abaxis API provides 3 modes to communicate to it in:
- Client Mode: in this mode API client communicates to a server which provides an ability to place orders and to receive results without knowing any specific about analyzers details. Practice is specified by its own credentials and client_id value.
- Server Mode: in this mode API client can communicate to analyzers directly.
- Partner Mode: PIMS communicates to Abaxis API as a platforms, using own credentials. Clinic is specified in each call to API by client_id value.

Rhapsody uses Partner Mode, i.e. each call contains Rhapsody username and password and clinic's client ID value.

Ordering flow

Ordering is performed in one step. When user presses “Place order“, a new order is prepared and sent to Abaxis API. It is not possible to modify it after placing.

 

 

Order Updates and Results receiving

Abaxis connector tries to get order updates every 30 seconds. The response contains updates for all orders of all practices. Each order update contains clinic_id value which identifies clinic to which order belongs.

Order results are received in batches, i.e. there could be several results available. Abaxis connector processes them one-by-one and acknowledges each order update individually.

When order update contains “COMPLETED“ status, antech connector tries to get order results for such order and process them.
